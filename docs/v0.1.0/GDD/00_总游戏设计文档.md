## 1. 执行摘要

### 1.1 游戏标识

| 项目 | 内容 |
| --- | --- |
| 游戏名称 | 女友（Girlfriends） |
| 类型 | Minecraft 模组 / 全年龄美少女养成 |
| 平台 | Minecraft Java Edition（Forge/NeoForge） |
| 目标版本 | v0.1.0 |
| 目标受众 | 喜欢 Minecraft 生存建设、角色陪伴与轻剧情推进的玩家 |
| 游戏时长 | 单角色主线约 15~25 小时；全角色约 80~120 小时 |

### 1.2 电梯简版

《女友》是一款面向 Minecraft 生存模式的全年龄功能性美少女养成模组。玩家在探索世界时自然遇见五位各具主题的角色，通过赠礼、委托与陪伴逐步建立羁绊，解锁跟随祝福、家园同居与终章专属功能奖励，让角色成为原版生存、建设、战斗与探索循环的长期增强者而非替代者。

### 1.3 独特卖点

1. **陪伴增强原版**：角色不替玩家玩游戏，而是让原版循环更有动力和温度
2. **五位差异化角色**：每位角色对应不同原版玩法维度（自然、航海、采矿、战斗、探索），形成互补而非重叠的玩法价值
3. **轻叙事重反馈**：拒绝大段文本，通过短句任务卡、行为反馈与环境变化推进剧情
4. **多人轻竞争**：可竞争、可转移、可协助的攻略关系规则，为服务器增添社交张力
5. **终章后长线循环**：主线完成后通过随机委托、礼物、跟随与家园互动持续维持关系体验

## 2. 设计支柱

### 2.1 五大设计支柱

#### 支柱一：陪伴增强，而非替代

> 角色让原版变得更好，而不是替玩家完成原版。

- 角色跟随提供祝福 Buff，但不自动替玩家挖矿、打怪、建造
- 终章奖励强化特定玩法方向，但保留玩家亲自操作的必要性
- 不做全自动资源采集、自动战斗或自动建造

#### 支柱二：关系可感知，而非可计算

> 玩家感受到羁绊在成长，而不是在刷数值。

- 模糊进度条 + 阶段文本反馈 + 环境变化来传达关系成长
- 委托推进伴随角色行为、庇护所外观与对话语气的渐变
- 不显示具体好感度数值，不做“刷到 X 点解锁 Y”的纯数值驱动

#### 支柱三：叙事轻量化，事件化表达

> 剧情通过行为和环境传达，而非文字堆砌。

- 委托任务卡（短句描述目标与动机）+ 聊天栏反馈 + 庇护所环境变化
- 角色行为变化（更频繁的互动、主动靠近、语气软化）推进叙事
- 不做大段对话框、过场动画或长篇文字剧情

#### 支柱四：主题差异化，玩法互补

> 五位角色 = 五种原版玩法的增强入口，不重叠、不冗余。

- 沫沫 → 自然 / 花草、渔溪 → 航海 / 钓鱼、梅疏 → 采矿 / 材料、晚萤 → 战斗 / 防护、幽若 → 末影 / 瞬移
- 每位角色的祝福、委托与终章奖励严格围绕核心主题
- 不做“万能角色”或主题重叠的角色

#### 支柱五：可竞争的羁绊，可预期的规则

> 多人场景下，攻略规则清晰、公平、可预期。

- 绑定、动摇期、转移、亲密锁定规则明确且可被玩家理解
- 协助机制让非绑定玩家也能参与内容，但奖励归属清晰
- 不做混乱的“先到先得”或无法被玩家理解的隐藏竞争逻辑

## 3. 核心玩法循环

### 3.1 总体循环图

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│   探索世界 ──→ 发现角色与庇护所 ──→ 赠礼 & 互动                  │
│       ↑                                  ↓                       │
│       │                          好感度提升                       │
│       │                                  ↓                       │
│       │                    ┌────── 解锁委托 ──────┐              │
│       │                    ↓                       ↓              │
│       │              随机委托                  固定委托           │
│       │              (日常循环)               (主线推进)          │
│       │                    ↓                       ↓              │
│       │              小额奖励            ┌─ 关系阶段推进          │
│       │                                  │─ 庇护所环境变化        │
│       │                                  │─ 角色行为变化          │
│       │                                  └─ 解锁新系统            │
│       │                                           ↓              │
│       │                              跟随祝福 ──→ 家园同居       │
│       │                                  ↓                       │
│       └──────────── 终章奖励 ←────────── 固定委托完结             │
│                         ↓                                        │
│                  终章后长线循环                                    │

│                                                                  │

```

### 3.2 玩家旅程

| 阶段 | 好感度 | 玩家行为 | 角色反馈 |
| --- | --- | --- | --- |
| 初见 | 0 | 在世界中探索，发现角色庇护所 | 角色在庇护所周边进行日常活动，可交互但保持距离 |
| 试探 | 0~100 | 赠送礼物、完成简单随机委托 | 接受礼物，给出简短反馈，偶尔发布小委托 |
| 建立信任 | 100~200 | 接到第一条固定委托，开始了解角色愿望 | 开始谈论自己的兴趣与愿望方向 |
| 稳定合作 | 200~500 | 推进第 2~4 条固定委托，深度参与角色生活 | 庇护所开始发生变化，语气明显更信任玩家 |
| 关系深化 | 500~700 | 推进第 5~7 条固定委托，面对角色核心愿望 | 角色表现出对玩家的依赖与在意，绑定竞争规则开启 |
| 亲密确认 | 700~800 | 完成第 8 条固定委托，正式确认亲密关系 | 角色可跟随并提供祝福，语气亲密而稳定 |
| 亲密深化 | 800~900 | 推进第 9 条固定委托，愿望接近完成 | 角色愿望基本实现，庇护所全面升级 |
| 家园同居 | 900~1000 | 完成第 10 条固定委托，邀请入住 | 角色入住玩家家园，形成长期生活感 |
| 终章之后 | 1000 | 随机委托 + 赠礼 + 陪伴 + 家园互动 | 持续发布随机委托，家园日常互动 |

### 3.3 核心体验目标

| 序号 | 体验目标 | 实现方式 |
| --- | --- | --- |
| 1 | 自然发现角色的惊喜感 | 角色生成于特定群系/维度，无主动定位系统，依赖玩家探索 |
| 2 | 关系成长的渐进感 | 六阶段好感度系统 + 环境变化 + 行为渐变 + 模糊进度条 |
| 3 | 角色价值的差异化感知 | 五位角色对应五种原版玩法维度，祝福与终章奖励各具特色 |
| 4 | 愿望实现的参与感 | 10 条固定委托串联完整角色弧线，玩家是愿望的共同完成者 |
| 5 | 长期陪伴的生活感 | 家园同居 + 持续随机委托 + 每日互动形成稳定的后期循环 |

## 5. 委托系统设计

### 5.1 固定委托设计说明

每位角色拥有 10 条固定委托，按好感度阶段分布如下：

| 委托序号 | 阶段 | 好感度区间 | 叙事功能 |
| --- | --- | --- | --- |
| 第 1 条 | 熟悉 | 100~200 | 建立主线方向与初步信任 |
| 第 2~4 条 | 信赖 | 200~500 | 深化角色习惯、能力与愿望基础 |
| 第 5~7 条 | 爱慕 | 500~700 | 推动角色面对长期愿望 |
| 第 8 条 | 爱慕→亲密 | 700 | 显式确认亲密关系（需关键交互触发） |
| 第 9 条 | 亲密 | 700~900 | 强化共同行动与愿望收束 |
| 第 10 条 | 亲密→家园 | 900~1000 | 家园前置与终章衔接 |

固定委托采用单人接取制。接取后一直保留直到完成，不会失败或过期。奖励与推进资格归属于接取者，其他玩家可自由协助。

## 6. 随机委托系统设计

### 6.1 随机委托设计原则

1. **不承担主线叙事**：随机委托是日常循环内容，不推进角色核心愿望
2. **按好感阶段逐步解锁**：更高阶段解锁更高价值 / 更高难度的随机委托模板
3. **5~10 游戏日随机刷新**：保持适度的间歇性，不让玩家感到被频繁催促
4. **自动过期无惩罚**：未完成的随机委托在下次刷新前自动过期，不给玩家压力
5. **小额奖励为主**：好感度 +5~15 + 主题资源，不投放破坏原版平衡的稀有物品

### 6.2 随机委托刷新机制

| 参数 | 设定 |
| --- | --- |
| 刷新间隔 | 5~10 个游戏日之间随机 |
| 过期条件 | 下次刷新前未完成则自动过期 |
| 过期惩罚 | 无 |
| 刷新逻辑 | 每次刷新从当前阶段已解锁的模板中随机抽取 1 类，并随机生成该类下的具体目标 |
| 连续刷新 | 同一模板不会连续出现两次（连续刷新同一模板概率为 0） |
| 委托槽占用 | 若当前有未完成的随机委托或固定委托，不刷新新的随机委托 |

## 7. 关系系统设计

### 7.1 好感度阶段与解锁总表

| --- | --- | --- | --- | --- |

### 7.2 好感度获取途径与数额

| 途径 | 单次基础值 | 频率限制 | 说明 |
| --- | --- | --- | --- |
| 最喜好礼物 | +5 × √((16 - 当日已增量) / 16) | 每日上限 16 点（总量） | 公式使边际收益递减 |
| 喜好礼物 | +4 × √((16 - 当日已增量) / 16) | 同上 | 同上 |
| 普通接受礼物 | +2 × √((16 - 当日已增量) / 16) | 同上 | 同上 |
| 厌恶礼物 | -2 × √((16 - 当日已增量) / 16) | 同上（计入日上限） | 同上，且为负值 |
| 固定委托完成 | +50 | 每条 1 次 | 主线推进，10 条共 +500 |
| 随机委托完成 | +8~15 | 每 5~10 日 1 次 | 小额循环收益 |
| 家园相处 | +2 / 游戏日 | 每日最多 1 次 | 需在双人床 20 格内 |
| 角色死亡 | 清零 | 无 | 好感度、委托、绑定、家园全部清零 |

| 事件 | 下降值 | 说明 |
| --- | --- | --- |

| 家园争执目击 | -3~-5 | 每玩家每日最多 1 次 |
| 角色死亡 | 清零 | 全部关系状态重置 |

## 8. 祝福与协助系统设计

### 8.1 祝福效果详表

| 角色 | 祝福名称 | 效果类型 | 效果描述 | 设计意图 |
| --- | --- | --- | --- | --- |
| 沫沫 | 自然的宽恕 | 被动 | 玩家不会吸引蜘蛛、熊猫、狼、北极熊、羊驼、山羊、蜜蜂与疣猪兽的仇恨 | 让玩家在自然环境中更从容地探索与建设，但不消除所有危险 |
| 渔溪 | 潮汐同行 | 被动 | 玩家划船速度 +50%；钓鱼时有 25% 概率获得双倍收益 | 强化水上旅行效率与渔获收益，与航海主题一致 |
| 梅疏 | 大地馈赠 | 被动 | 玩家挖掘矿物时有 25% 概率额外掉落对应矿物 | 让采矿过程更有期待感，但不替代玩家的采矿行为 |
| 晚萤 | 烈焰守护 | 被动 | 玩家近战伤害 +40%；受到的伤害 -25% | 显著增强近战生存能力，体现战斗支援定位 |
| 幽若 | 虚空回响 | 被动 | 玩家投出的末影珍珠有 25% 概率不消耗 | 鼓励更多空间移动尝试，降低瞬移试错成本 |

### 8.2 跟随行为设计

| 角色 | 跟随距离 | 短距采集 | 战斗行为 | 特殊行为 |
| --- | --- | --- | --- | --- |
| 沫沫 | 3~8 格 | 目标方块在 2 格内时采集花草、树叶（极短距） | 不参与战斗 | 会主动靠近花朵和蜜蜂，偶尔向玩家抛出一朵花 |
| 渔溪 | 3~8 格 | 目标方块在 2 格内时采集钓鱼相关物品 | 不参与战斗 | 靠近水域时会面向水面，偶尔发出钓鱼的音效 |
| 梅疏 | 3~8 格 | 目标方块在 2 格内时采集矿石（极短距，仅地表矿） | 不参与战斗 | 靠近矿道或岩壁时会仔细观察，偶尔敲击方块 |
| 晚萤 | 2~6 格 | 不进行采集 | 主动攻击索敌玩家的敌对生物 | 保持警觉姿态，会挡在玩家与敌对生物之间 |
| 幽若 | 3~8 格 | 目标方块在 2 格内时采集末地相关样本 | 不参与战斗 | 偶尔抛出末影珍珠进行短距瞬移，跟随瞬移 |

### 8.3 跟随交互设计

| 交互项 | 说明 |
| --- | --- |
| 开始跟随 | 与角色交互选择“一起走吧”；仅亲密阶段以上可用 |
| 停止跟随 | 与角色交互选择“在这里等我”；或超过一定距离后角色自动返回庇护所 / 家园 |
| 跟随状态 | 角色在玩家周围 3~8 格内活动，不挡住玩家视线和操作 |
| 战斗响应 | 除晚萤外不主动参与战斗；角色受到攻击时会短暂后退 |
| 传送跟随 | 当玩家使用末影珍珠、下界传送门或末地传送门时，角色也会跟随传送 |
| 家园返回 | 当角色为家园伙伴时，停止跟随后返回家园而非庇护所 |

## 9. 家园系统设计

### 9.1 入住流程

```
好感度 ≥ 900
    ↓
完成第 10 条固定委托（家园前置）
    ↓
玩家家园中有双人床
    ↓
与角色交互 → 选择“邀请入住”
    ↓
角色入住家园，成为家园伙伴
    ↓
家园收益生效
```

### 9.2 家园收益详表

| 收益项 | 效果 | 条件 | 频率 |
| --- | --- | --- | --- |
| 生命恢复 | 每 10 秒恢复 1 点生命值 | 玩家与家园伙伴同时处于双人床 20 格内 | 持续触发 |
| 好感增长 | +2 好感度 | 玩家与家园伙伴同时处于双人床 20 格内 | 每游戏日最多 1 次 |
| 陪伴氛围 | 家园伙伴在家中活动、布置、偶尔与家具互动 | 家园伙伴处于入住状态 | 持续 |

### 9.3 家园更换规则

| 步骤 | 说明 |
| --- | --- |
| 1. 解除绑定 | 与当前家园伙伴交互，选择“解除同居关系” |
| 2. 关系保留 | 解除绑定后好感度不变，角色仍为亲密阶段 |
| 3. 角色返回 | 解除绑定后，角色返回其原始庇护所 |
| 4. 冷却期 | 解除绑定后 3 个游戏日内不能邀请新家园伙伴 |
| 5. 重新邀请 | 冷却期后，可邀请其他符合条件的角色入住 |
| 6. 再次邀请原角色 | 若想再次邀请同一角色，需重新提升好感度至 900（通常已满足） |

### 9.4 家园冲突事件设计

| 维度 | 描述 |
| --- | --- |
| 触发条件 | 玩家已与 ≥1 名非家园伙伴角色保持亲密关系，并将其中一位带入家园 |
| 目击范围 | 家园伙伴在双人床 20 格内目击玩家与来访亲密角色互动 |
| 表现形式 | 聊天栏出现家园伙伴的短句（如沫沫：“……她也是来看花的吗？”，晚萤：“你带别人来家里？”等） |
| 效果 | 家园伙伴与来访角色各对玩家扣除 3~5 点好感度 |
| 频率限制 | 每位玩家每游戏日最多触发 1 次 |
| 不触发情况 | 来访角色非亲密关系（仅信赖或以下）；家园伙伴不在目击范围内 |

## 10. 多人系统设计

### 10.1 绑定与竞争规则流程

```
初始状态：角色不绑定任何玩家，每位玩家独立计算好感度
    ↓
玩家 A 对角色好感度达到 500（爱慕阶段）
    ↓
角色不再向其他玩家发布固定委托，不再接受其他玩家礼物
    ↓
其他玩家仍可通过随机委托缓慢增长好感度（随机委托刷新较长）
    ↓
┌─ 情况一：无其他玩家超越 ─→ 玩家 A 保持绑定，继续推进
│
└─ 情况二：玩家 B 好感度超越玩家 A
        ↓
    进入 3 游戏日动摇期
        ↓
    系统向玩家 A 发出预警
        ↓
    ┌─ 玩家 A 在动摇期内反超 ─→ 绑定维持，动摇结束
    │
    └─ 动摇期满，玩家 B 仍领先
            ↓
        绑定转移至玩家 B
        ↓
    玩家 A 已完成委托与好感度不清零
    角色后续固定委托由玩家 B 接取
    ↓
玩家 B 达到亲密（完成第 8 条委托）
    ↓
角色不再向其他任何玩家发布随机委托
其他玩家无法继续攻略该角色
```

### 10.2 多人协助规则

| 规则项 | 说明 |
| --- | --- |
| 委托协助 | 其他玩家可以帮助接取者完成委托目标（如交付材料、击杀目标等） |
| 奖励归属 | 委托奖励（好感度、资源、推进进度）仅归接取者 |
| 协助动机 | 协助者无直接利益，但可以通过协助间接影响关系（如加速固定委托推进、或拖慢） |
| 终章奖励 | 按玩家唯一原则发放，每位玩家对每位角色仅首次通关可领取一次功能奖励 |
| 协助限制 | 亲密确认委托（第 8 条）不允许其他玩家参与 |

### 10.3 多人场景边界规则

| 场景 | 规则 |
| --- | --- |
| 同时跟随 | 一名角色同一时间只能跟随一位玩家 |
| 家园伙伴 | 一名角色同一时间只能是一位玩家的家园伙伴 |
| 同时交互 | 当角色正在与玩家 A 交互时，玩家 B 的交互请求排队 |
| 庇护所可见 | 庇护所的成长变化对所有玩家可见（环境变化是客观存在的） |
| 角色记忆 | 角色死亡后，所有玩家的关系数据清零（共同遗忘） |

## 11. 终章奖励设计

### 11.1 终章奖励设计原则

1. **体现角色主题**：每位角色的终章奖励严格围绕其核心玩法方向
2. **强化但不替代**：奖励增强玩家的相关玩法能力，但不自动完成原版行为
3. **唯一发放**：每位玩家对每位角色仅首次完成第 10 条固定委托时获得功能奖励
4. **角色死亡不影响已获得奖励**：终章奖励相关物品、方块、配方等永久保留

### 11.2 终章奖励方向详表

| 角色 | 奖励主题 | 奖励方向描述 | 设计意图 |
| --- | --- | --- | --- |
| 沫沫 | 温室与自然 | 解锁可建造的温室结构方块 / 配方，花类与蜂类生态的持续加成，自然装饰物制作权限 | 让玩家能在家园中复现并持续受益于沫沫的园艺成果 |
| 渔溪 | 远航与钓鱼 | 解锁高级船只或航海辅助物品，钓鱼效率或稀有渔获概率永久小幅提升，海洋探索便利性增强 | 让远航与钓鱼在终章后依然有持续吸引力 |
| 梅疏 | 合金与采矿 | 解锁特殊合金配方或工具，采矿效率或深层矿物发现率小幅提升，材料加工便利性增强 | 让合金研究的结果以可用的功能性物品形式回馈玩家 |
| 晚萤 | 战斗与防护 | 解锁特殊战斗物品或护甲配方，近战战斗效率小幅永久提升，防护能力增强 | 让晚萤的守护精神以战斗增益形式持续陪伴玩家 |
| 幽若 | 瞬移与末影 | 解锁可放置的回归锚点方块或瞬移便利物品，末影珍珠使用效率提升，末地探索便利性增强 | 让幽若的研究成果以可使用的空间移动工具形式回馈玩家 |

## 12. UI / UX 设计方向

### 12.1 角色交互界面

| 界面区域 | 内容 |
| --- | --- |
| 顶部 | 角色名称 + 模糊好感度进度条（无具体数值） |
| 左侧 | 角色当前状态（待机 / 跟随 / 家园）+ 当前委托信息 |
| 中央 | 交互选项区：赠礼、委托、跟随 / 停留、家园邀请、偏好记录 |
| 右侧 | 已知礼物偏好记录（渐进揭示，只显示已确认的偏好） |
| 底部 | 角色短句反馈区（根据当前状态和好感度变化动态更新） |

### 12.2 角色头顶标识

| 标识类型 | 图标 | 含义 |
| --- | --- | --- |
| 无标识 | 无 | 当前没有可接取的委托 |
| 随机委托标识 | 浅绿色问号图标 | 当前有一条随机委托可接取 |
| 固定委托标识 | 金色感叹号图标 | 当前有一条固定委托可接取 |
| 委托进行中 | 灰色沙漏图标 | 已有玩家接取了当前委托（但尚未完成） |
| 亲密关系标识 | 粉色小心形 | 该角色已与某位玩家确认亲密关系（对其他玩家不可攻略） |

### 12.3 聊天栏反馈规范

| 情境 | 反馈方式 | 示例 |
| --- | --- | --- |
| 赠送最喜好礼物 | 角色头顶冒出心形粒子 + 聊天栏短句 | 沫沫：“这是我最喜欢的……谢谢你记得。” |
| 赠送厌恶礼物 | 角色摇头 + 聊天栏短句 | 晚萤：“这种东西……下次不要给了。” |
| 委托完成 | 角色头顶冒出星星粒子 + 聊天栏短句 | 渔溪：“有你在真好。” |
| 礼物日上限已达 | 聊天栏提示 | 角色拒绝接取，聊天栏：“你今天已经送我很多礼物了……” |
| 家园争执 | 聊天栏短句（感叹号前缀） | 梅疏：“……你带别人来家里了？” |
| 关系阶段升级 | 聊天栏短句 + 粒子效果 | 幽若：“我想把坐标设在你身边。” |
| 绑定动摇预警 | 聊天栏提示（警告色） | 系统提示：“晚萤对另一人的好感正在接近你的水平……” |

## 13. 角色死亡与重生设计

### 13.1 死亡后果

| 后果项 | 说明 |
| --- | --- |
| 物品遗失 | 角色死亡后失去全部随身物品 |
| 关系清零 | 对所有玩家的好感度、委托进度、绑定关系、家园状态全部清零 |
| 记忆清零 | 角色失去所有“记忆”——相当于一个全新的角色实例 |
| 庇护所保留 | 庇护所结构保留，但成长变化重置为基础状态 |
| 玩家资产保留 | 玩家已获得的终章奖励、物品、配方等不回收 |
| 已完成委托 | 玩家的“已完成固定委托”记录保留，但角色端进度重置 |

### 13.2 重生规则

| 规则项 | 说明 |
| --- | --- |
| 重生位置 | 距离死亡位置最近的该角色已生成庇护所 |
| 重生条件 | 必须存在至少一个已生成的该角色庇护所 |
| 无可用庇护所 | 角色暂不重生，待后续世界探索生成新庇护所后出现 |
| 维度限制 | 晚萤仅在下界重生，幽若仅在末地重生，其余三位仅在主世界重生 |
| 重生延迟 | 角色死亡后立即尝试重生（若条件满足则即刻在对应庇护所出现） |

## 15. 风险与应对

| 风险 | 严重度 | 应对策略 |
| --- | --- | --- |
| 角色 AI 行为过于简单，缺乏“生命感” | high | 通过日常行为模式 + 环境变化 + 短句反馈来弥补 AI 深度不足，优先保证“可感知的变化” |
| 固定委托玩法单一（采集 / 交付过多） | medium | 在委托中融入陪伴型目标（陪同钓鱼、共同战斗、一起航行），增加互动感 |
| 终章后内容空洞 | high | 随机委托 + 家园日常 + 多人竞争构成终章后循环；后续版本可增加新角色 |
| 多人竞争导致负面体验 | medium | 动摇期机制给原绑定者反超机会；终章奖励按玩家唯一原则发放，不因绑定转移而取消 |
| 角色死亡重置引发挫败感 | medium | 玩家资产不回收；庇护所结构保留；且角色死亡为低频事件 |
| 五位角色开发量过大 | high | 建议按角色逐个实现，首位角色（建议沫沫）完成后即进入可玩验证阶段，验证通过后再扩展其余四位 |

## 17. 术语表

| 术语 | 定义 |
| --- | --- |
| 固定委托 | 承担角色主线推进的 10 条固定任务，接取后不失效不失败 |
| 随机委托 | 承担日常循环的刷新型小任务，5~10 日刷新，可过期无惩罚 |
| 委托槽 | 每位角色同时只能发布 1 个委托的机制 |
| 模糊进度条 | 不显示具体数值的好感度可视化方式 |
| 亲密确认 | 好感度 700 时通过第 8 条固定委托显式确立亲密关系 |
| 家园伙伴 | 被邀请入住玩家家园的角色，同一时间只能有 1 位 |
| 绑定 | 好感度达到爱慕阶段后，角色仅接受该玩家的固定委托与礼物 |
| 动摇期 | 其他玩家好感度超越当前绑定玩家后的 3 游戏日竞争窗口 |
| 终章奖励 | 完成第 10 条固定委托后的永久功能性奖励，每位玩家对每位角色仅首次获得 |
| 庇护所 | 角色生成时自带的主题轻量结构，承担发现、重生与主题展示功能 |
